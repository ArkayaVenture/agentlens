# Screenshots

Drop product screenshots here so the README and landing page render them.

**Required for the README hero:** `overview.png` — the dashboard **Overview** tab.

## How to capture (clean, high-res)
1. Make sure the service is running: open http://127.0.0.1:7717/ .
2. Use the **Dark** theme (default) for the hero; optionally capture a Light variant.
3. Recommended shots:
   - `overview.png` — Overview (KPIs + provenance + token chart). **Used in README.**
   - `provenance.png` — the 4-way provenance breakdown expanded.
   - `session-drilldown.png` — a session drawer showing the tool timeline.
   - `topology.png` — the capability graph.
   - `ultracode.png` — the ULTRACODE strategy view.
4. Export at ~1600px wide (Retina), PNG. Keep file sizes < 600 KB (use pngquant/tinypng).

## Optional: animated demo
A short `demo.gif` (≤ 8 s, < 5 MB) of drilling Overview → session → tool call makes the README and
social posts far more compelling. Reference it in the README hero in place of `overview.png` if you have one.

> Tip: the landing page (`marketing/landing/index.html`) ships with an inline SVG mock, so it needs
> no screenshot to look good — but a real screenshot in the README builds the most trust.
